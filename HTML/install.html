 Community Linux is is easy to use. It boots directly from a USB drive, and presents a graphical user interface without any user intervention. One may run applications, surf the net, administer systems without any further setup or installation procedure(s).

 When prompted for passwords, ComLin's default password for the initial 'user' is 'user1234' and for the 'root' user is 'admin1234' (without the quotes). One needs to change these passwords before ComLin can be deployed as a personal workstation, or connected to a network. The passwords can be changed from the command line or from a small password changing GUI presented on the desktop or your favorite password changing program.

  Additional security measures. ComLin contains an encrypted folder under '~/encrypted/Documents/'. The folder is empty, and the initial password to the folder is '1234'. Changing the password to the folder can be done from the provided 'cryptkeeper' utility from the main panel. (icon that looks like a little padlock) After changing the password to the encrypted folder, the contents of the folder will only be accessible after successfully entering the new password.

    NOTE: Industrial strength encryption, your data cannot be recovered without the password.

  One can create new users in ComLin with any of the traditional GUI or command line tools like 'useradd'. However, there is no need for additional Linux system users. Community Linux deploys a single user policy, under which, several users can have private data in the 'encrypted' folder. This policy serves for simplicity, openness, and if needed, personal privacy.

  No measures have been taken to enforce the single user policy, so one is free to take advantage of the multi-user nature of Linux. Naturally, this assumes (some) system administrators knowledge, which is contrary to ComLin's original premise.
