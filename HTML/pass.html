Default passwords:

  Initially, Community Linux runs as 'user' and it has the default password as 'user1234'. (without the quotes) The default 'user' is the sudoers file, so all administarion tasks can be done as 'user'. Community Linux also has an enabled 'root' user, with a password of 'admin1234'. Before you create any personal documents, or connect to any network, make sure you change these passwords. The passwords can be changed by any of the traditional UNIX tools like 'passwd' or any GUI password changing utility. There are two icons on the desktop for changing the passwords. It is worthy to note, that we switched cracklib off, so the passwd utility accepts simpler passwords. But just because the system is more permissive, you may still have a password as complex as you wish.

Encryption subsystem:

    Community Linux is also equipped with a state of the art encryption subsystem. We created a default setting to make it easy to use. The default setting can be enabled by clicking on the padlock icon, (on the upper toolbar, on the right side) enabling the encrypted/documents directory, and entering the default password '1234' Before you create any personal documents, or connect to any network, make sure you change the encryption password.

System firewall:

    The system firewall is enabled, and some of the standard ports are open. (SSH, HTTP ... etc) If you changed the passwords before you connected to a network, these settings should be fine. If higher security is desired, please refer to the System Administration Manual or consult a system administrator.

System lock down:

    Community Linux is designed to be a personal class system. However, it adheres to UNIX standards, so all the security measures of Linux are available. (selinux, polkit) For further details, please refer to the System Administration Manual or consult a system administrator.
